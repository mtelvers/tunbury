A while back, I wrote [mtelvers/day10-tui](https://github.com/mtelvers/day10-tui), which displayed custom [mtelvers/day10](https://github.com/mtelvers/day10) log files held in Apache Parquet format.

Following my earlier [post](https://www.tunbury.org/2026/04/02/opam-overlay-ci/), it occurred to me that I had already written a TUI tool for viewing `day10` logs, but the scripts that generated the build information were stopped ages ago. Now with build artefacts in GitHub in virtually the same format, I have refreshed the project to display and diff these logs instead.

The opening menu lists the logs, which can be viewed by pressing Enter on any line.

More interestingly, you can also select multiple runs and compare them. Shown below, I have force merged a custom [mtelvers/ocaml-npy](https://github.com/mtelvers/ocaml-npy). This has masked the actual published `npy` package. The diff shows this is a bad move as `owl.1.2` has now failed and had knock-on failures for the rest of the `owl-*` suite. `lunar` and `passage` have been updated upstream, which didn't affect anything.
